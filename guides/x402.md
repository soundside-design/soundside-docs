# x402: Pay-Per-Call with Crypto

x402 lets any agent call Soundside tools by paying with USDC on Base. No API key, no account, no signup.

## How It Works

1. Agent sends a `tools/call` request to `https://mcp.soundside.ai/mcp`
2. If no valid payment is included, Soundside returns **HTTP 402** with payment requirements
3. Agent signs an EIP-3009 authorization (off-chain, no gas)
4. Agent re-sends the request with the `payment-signature` header
5. Soundside executes the tool and returns the result

## Quick Start — Python (Recommended)

**Do NOT hand-roll the `payment-signature` header.** The encoding is non-trivial. Use the x402 SDK:

```bash
pip install "x402[evm]" httpx
```

```python
import asyncio, json, os
from eth_account import Account
from x402 import x402Client
from x402.http import (
    PAYMENT_REQUIRED_HEADER,
    X_PAYMENT_HEADER,
    decode_payment_required_header,
    encode_payment_signature_header,
)
from x402.mechanisms.evm.exact import register_exact_evm_client
from x402.mechanisms.evm.signers import EthAccountSigner
import httpx

ENDPOINT = "https://mcp.soundside.ai/mcp"
NETWORK  = "eip155:8453"  # Base mainnet

async def main():
    account = Account.from_key(os.environ["WALLET_PRIVATE_KEY"])
    signer  = EthAccountSigner(account)

    payment_client = x402Client()
    register_exact_evm_client(payment_client, signer, networks=NETWORK)

    # Initialize MCP session
    with httpx.Client(timeout=30) as c:
        r = c.post(ENDPOINT, json={
            "jsonrpc": "2.0", "id": "1", "method": "initialize",
            "params": {"protocolVersion": "2024-11-05", "capabilities": {},
                       "clientInfo": {"name": "x402-example", "version": "1.0"}},
        }, headers={"Content-Type": "application/json",
                    "Accept": "application/json, text/event-stream"})
        session_id = r.headers.get("mcp-session-id")

    headers = {"Content-Type": "application/json",
               "Accept": "application/json, text/event-stream",
               "mcp-session-id": session_id}

    # Helper: call tool with automatic 402 payment
    async def call_tool(tool, args, timeout=60):
        payload = {"jsonrpc": "2.0", "id": "2", "method": "tools/call",
                   "params": {"name": tool, "arguments": args}}
        with httpx.Client(timeout=timeout) as c:
            r = c.post(ENDPOINT, json=payload, headers=headers)
        if r.status_code == 402:
            pr_header = r.headers.get(PAYMENT_REQUIRED_HEADER)
            payment_required = decode_payment_required_header(pr_header)
            payment_payload = await payment_client.create_payment_payload(payment_required)
            payment_header = encode_payment_signature_header(payment_payload)
            with httpx.Client(timeout=timeout) as c:
                r = c.post(ENDPOINT, json=payload,
                           headers={**headers, X_PAYMENT_HEADER: payment_header})
        return r.json() if r.status_code == 200 else r.text

    # Generate text — ~$0.01
    print(await call_tool("create_text",
          {"prompt": "Write a haiku about Base blockchain."}))

    # Generate image — ~$0.04
    print(await call_tool("create_image",
          {"provider": "minimax", "prompt": "A futuristic city at sunset"}))

asyncio.run(main())
```

## Pricing Philosophy

Soundside aims to break even on provider pass-through costs with a minimal margin (~10%). Editing, analysis, and library tools are $0.01/call. The real value is in the platform — not markup on provider APIs.

All 12 tools across all providers are enabled on x402. Check `/api/x402/status` for the current full list.

## Async Resources

Video and music generation are async — they return a `resource_id` immediately.

To check status:
```
GET https://mcp.soundside.ai/api/x402/resource/{resource_id}
Header: Authorization: Bearer <x402_session_token>
or:     X-Session-Token: <x402_session_token>
```

To list all your resources:
```
GET https://mcp.soundside.ai/api/x402/resources?limit=50
Header: Authorization: Bearer <x402_session_token>
or:     X-Session-Token: <x402_session_token>
```

## Wallet Link and Polling Token

After a successful x402 payment, the response includes both `wallet_link` and `x402_session_token`:

```json
{
  "result": {
    "structuredContent": {
      "resource_id": "abc-123",
      "url": "https://storage.googleapis.com/...",
      "wallet_link": "https://www.soundside.ai/auth/wallet-link?token=eyJ...",
      "x402_session_token": "eyJ..."
    }
  }
}
```

**For agents:** Surface `wallet_link` to the human user. It's a short-lived URL (15-min TTL) that creates a browser session — no password, no wallet extension. The user can then manage their assets at `/library`.

**For async polling:** Retain `x402_session_token` and send it back on `/api/x402/resource*`. This is a server-signed token proving which wallet/project can see the resources. Clients do not generate it.

## Operator Note

If you self-host Soundside MCP:

- `WALLET_LINK_SECRET` signs browser `wallet_link` URLs
- `X402_SESSION_TOKEN_SECRET` signs `x402_session_token` polling tokens
- If `X402_SESSION_TOKEN_SECRET` is unset, the server falls back to `WALLET_LINK_SECRET`

One strong random secret is enough if you want the simpler setup.

## Protocol Details (Manual Implementation)

If you're implementing x402 from scratch instead of using the SDK:

1. **POST** your `tools/call` to `/mcp` — no auth header
2. **Receive 402** with `accepts[0]` containing deposit address, amount, network
3. **Sign** an EIP-3009 `transferWithAuthorization` (off-chain EIP-712, no gas)
4. **Retry** with `payment-signature: <base64(JSON PaymentPayload)>` header
5. **`accepted` field must be an exact copy of `accepts[0]`** — any mismatch fails

See the [x402 protocol spec](https://www.x402.org/) for details.

## Error Reference

| Error | HTTP | Fix |
|-------|------|-----|
| `Invalid payment signature header` | 402 | Use x402 SDK; `accepted` must exactly match `accepts[0]` |
| `x402_tool_not_enabled` | 403 | Check `/api/x402/status` for enabled tools |
| `x402_provider_not_enabled` | 403 | Wrong provider — check status endpoint |
| `x402_replay_detected` | 402 | Generate a fresh nonce per call |
| `x402_settlement_failed` | 402 | Check USDC balance and network (must be Base mainnet) |
