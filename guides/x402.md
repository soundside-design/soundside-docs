# x402: Pay-Per-Call with Crypto

x402 lets any agent call Soundside tools by paying with USDC on Base. No API key, no account, no signup.

## How It Works

1. Agent sends a `tools/call` request to `https://mcp.soundside.ai/mcp-x402`
2. If no valid payment is included, Soundside returns **HTTP 402** with payment requirements
3. Agent signs an EIP-3009 authorization (off-chain, no gas)
4. Agent re-sends the request with the `payment-signature` header
5. Soundside executes the tool and returns the result

## Quick Start — Python (Recommended)

**Do NOT hand-roll the `payment-signature` header.** The encoding is non-trivial. Use the x402 SDK:

```bash
pip install "x402[evm]" mcp
```

```python
import asyncio, os
from eth_account import Account
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client
from x402 import x402ClientAsync
from x402.mechanisms.evm.exact import register_exact_evm_client
from x402.mechanisms.evm.signers import LocalAccountEVMSigner
from x402.mcp import x402MCPClient

ENDPOINT = "https://mcp.soundside.ai/mcp-x402"
NETWORK  = "eip155:8453"  # Base mainnet

async def main():
    account = Account.from_key(os.environ["WALLET_PRIVATE_KEY"])
    signer  = LocalAccountEVMSigner(account)

    payment_client = x402ClientAsync()
    register_exact_evm_client(payment_client, signer, networks=NETWORK)

    async with streamablehttp_client(ENDPOINT) as (read, write, _):
        async with ClientSession(read, write) as session:
            await session.initialize()

            # x402MCPClient intercepts 402, signs, and retries automatically
            x402_session = x402MCPClient(session, payment_client, auto_payment=True)

            # Generate text — ~$0.01
            result = await x402_session.call_tool(
                "create_text",
                {"prompt": "Write a haiku about Base blockchain."}
            )
            print(result.content[0].text)

            # Generate image — ~$0.04
            result = await x402_session.call_tool(
                "create_image",
                {"provider": "minimax", "prompt": "A futuristic city at sunset, cyberpunk style"}
            )
            print(result.content[0].text)  # Contains resource_id + signed URL

asyncio.run(main())
```

## Pricing Philosophy

Soundside aims to break even on provider pass-through costs with a minimal margin (~10%). Editing, analysis, and library tools are $0.01/call. The real value is in the platform — not markup on provider APIs.

All 12 tools across all providers are enabled on x402. Check `/api/x402/status` for the current full list.

## Async Resources

Video and music generation are async — they return a `resource_id` immediately.

Header: x402-wallet: <your-wallet-address>

```
GET https://mcp.soundside.ai/api/x402/resources?limit=50
Header: x402-wallet: <your-wallet-address>
```

## Wallet Link — Browser Access

After a successful x402 payment, the response includes a `wallet_link` URL:

```json
{
  "result": {
    "structuredContent": {
      "resource_id": "abc-123",
      "url": "https://storage.googleapis.com/...",
      "wallet_link": "https://www.soundside.ai/auth/wallet-link?token=eyJ..."
    }
  }
}
```

**For agents:** Surface `wallet_link` to the human user. It's a short-lived URL (15-min TTL) that creates a browser session — no password, no wallet extension. The user can then manage their assets at `/library`.

## Protocol Details (Manual Implementation)

If you're implementing x402 from scratch instead of using the SDK:

1. **POST** your `tools/call` to `/mcp-x402` — no auth header
2. **Receive 402** with `accepts[0]` containing deposit address, amount, network
3. **Sign** an EIP-3009 `transferWithAuthorization` (off-chain EIP-712, no gas)
4. **Retry** with `payment-signature: <base64(JSON PaymentPayload)>` header
5. **`accepted` field must be an exact copy of `accepts[0]`** — any mismatch fails

See the [x402 protocol spec](https://www.x402.org/) for details.

## Error Reference

| Error | HTTP | Fix |
|-------|------|-----|
| `Invalid payment signature header` | 402 | Use x402 SDK; `accepted` must exactly match `accepts[0]` |
| `x402_tool_not_enabled` | 403 | Check `/api/x402/status` for enabled tools |
| `x402_provider_not_enabled` | 403 | Wrong provider — check status endpoint |
| `x402_replay_detected` | 402 | Generate a fresh nonce per call |
| `x402_settlement_failed` | 402 | Check USDC balance and network (must be Base mainnet) |
